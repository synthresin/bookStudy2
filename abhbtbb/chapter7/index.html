딕셔너리와 해시

Hashing ( 해싱 )

  • hasing은 자료 구조에서 특정 값을 가장 신속하게 찾는 방법이다.
  • 해시 함수는 어떤 키에 해당하는 값의 주소를 테이블에서 찾아주는 함수라서 조회 속도가 빠르고 간단하다.
  • 가장 흔한 형태의 해시 함수는 일명 '루즈루즈 ( lose lose )' 해시 함수라고 하는데, 키룰 구성하는 아스키 값을 더한 것이다.

해시 테이블 만들기

  • 해시와 같은 자료구조는 배열을 사용해 나타낸다.
  • Hash Table처럼 Javascript Object도 name, value로 구성되어 있다.
  • Javascript Object도 해시 테이블이라고 볼수 있다. 이것은 Java의 HashMap, C++의 HashTable과 유사하다.

해시 테이블 간 충돌 해결

충돌이란?

  • 충돌 ( collision ) : 키는 다르지만 값은 동일한 경우 동일한 인덱스에 여러 값들이 들어가야 하는 상황.
  • 충돌이 발생했을 때 특별한 처리를 해줘야 한다. 이러한 처리 방법으로 체이닝, 선형 탐사, 이중 해싱이 있다.

체이닝

  • 체이닝 ( separate chaining ) 은 테이블의 인덱스 별로 연결리스트를 생성해 그안에 원소를 저장하는 기법이다.
  • 가장 간단하지만, HashTable 인스턴스 외에 메모리가 추가적으로 소용되는 단점이 있다.
  • 헬퍼 클래스로 링크드 리스트를 구현해 줘야 한다.
